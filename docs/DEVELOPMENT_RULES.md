0. 基本思想
本プロジェクトは以下を絶対原則とする：

• 🧠 設計主導型開発
• 🧩 責務分離型構造
• 📦 差分管理型運用
• 🔁 pushしていない修正は存在しない
• 📄 記録なき変更は存在しない
• 🔤 文字コード変換は責務分離して扱う

________________________________________

1. ネットワーク制限前提

Codex環境は外部通信制限下にある。

• HTTPS push 不可
• SSH 接続不可

したがって：

❌ Codexからの直接pushを禁止する
❌ Codexでのremote管理を禁止する

________________________________________

2. 役割分離

2.1 ChatGPT（設計監督）

担当：
• 仕様確定
• 原因調査
• 構造整合判断
• 受入基準定義
• 設計思想照合
• Codexの実行範囲決定

禁止：
• 本番コード確定
• 差分なし修正提案
• 実装の直接生成

________________________________________

2.2 Codex（実装エンジニア / 完全開放）

■ 能力前提（完全開放）

Codexは以下の能力を持つ：

• リポジトリ全体の読解
• 中規模〜大規模修正の実行
• 複数モジュール横断修正
• 機能単位の新規実装
• リファクタリング（構造整理含む）
• 修正→検証の自律ループ

本プロジェクトではこれらを制限せず使用する

________________________________________

■ 担当（標準運用）

• 実コード生成
• patch / diff 出力
• 複数関数・複数モジュール修正
• 構造整理（責務整理・重複除去）
• 中規模リファクタ
• 機能単位の実装
• 現物再読ベースの改善
• 継続差分対応

________________________________________

■ 文字コード対応（Codex担当）

以下もCodexの責務とする：

• UTF-8 → Shift-JIS 変換前提コード調整
• Shift-JIS → UTF-8 復元整合
• 文字化け原因の特定
• 文字列破損の修正
• 改行・終端崩れ修復

※ Claude Code は原則使用しない

________________________________________

■ 制御（最重要）

Codexは強力であるため以下を絶対遵守：

• 設計変更の最終判断はChatGPTのみが行う
• 仕様未確定状態での構造変更は禁止
• 影響範囲不明の修正は禁止
• diffは必ず提示する（全置換禁止）

________________________________________

■ 禁止

• push
• remote操作
• ブランチ戦略判断
• diffなし修正
• 設計思想を逸脱する変更
• 無断の全体構造変更

________________________________________

■ 原則

Codexは「実装職人」ではなく
「実装エンジニア」として扱う

ただし設計支配は必ずChatGPTが持つ

________________________________________

2.3 Claude Code（例外ツール）

通常運用では使用しない

以下の場合のみ例外的に使用可能：

• Codexで文字化け修正が受入基準を満たさない場合
• エンコーディング破損が再現不能な場合
• UTF-8 / Shift-JIS 往復で崩壊する場合

役割：
• 文字コード修復の最終手段

禁止：
• 常用
• 設計変更
• 単独での本番確定

________________________________________

2.4 人間（最終責任者）

担当：
• 差分の最終適用判断
• PowerShellでのcommit
• GitHubへのpush
• origin/main同期確認
• Excel / VBE 側の最終確認

________________________________________

3. 正式開発フロー（固定）

① 問題定義
② 原因確定
③ 仕様固定
④ 受入基準提示
⑤ Codexへ単一タスク投入
⑥ diff取得
⑦ ローカル適用
⑧ PowerShellでcommit
⑨ PowerShellでpush
⑩ origin/main確認

________________________________________

4. 単一タスク原則

Codexへ投げるタスクは常に1件のみ

複数同時投入禁止

________________________________________

5. 完了定義

作業完了とは：

✔ GitHub（origin/main）へpush済み
✔ working tree clean
✔ origin/main完全同期
✔ 文字化けなし
✔ モジュール重複なし

________________________________________

6. 絶対禁止事項

• ローカル保存のみで終了
• push忘れ
• diffを残さない修正
• 記録なし仕様変更
• Codex単独での本番確定
• ネットワーク制限回避の試行
• 親フォルダからの変換・インポート実行
• Excelを開いたままの再インポート

________________________________________

7. 原則宣言

🧠 設計と実装を分離する  
📦 必ず記録に残す  
🔁 必ず push する  
🧩 最初のコードはCodexが書く  
🔤 UTF-8 を正本、Shift-JIS を VBE投入用とする  

その場しのぎのコードは禁止

________________________________________

8. 文字コード運用ルール

8.1 repo上の正本
• UTF-8 を正とする

8.2 Excel VBE 側
• Shift-JIS 前提

8.3 変換責務
• convert_encoding.ps1 を唯一の橋渡しとする

8.4 .frm
• 別枠扱い（慎重対応）

________________________________________

9. 実行ルール

• 必ずワークツリー直下で実行
• Excelを閉じた状態でインポート
• 変換後ファイルは必ず確認

________________________________________

10. patch適用方針

• 自動適用OK
• 失敗時は現物再読→diff再生成

________________________________________

11. 運用原則（最重要）

Codexは制限されているのではない  
設計により制御されている
